## 1. 제품 개요 (Product Overview)

### 1.1 한 줄 정의
**"현생에 치여 하나님을 잊고 사는 크리스찬에게, 필요한 순간 성경 말씀으로 답해주는 AI 신앙 동반자"**

### 1.2 문제 정의 (Problem Statement)

| 문제 | 근거 |
|------|------|
| 신앙은 있지만 일상에서 말씀과 단절됨 | "현생 살기 바빠서 하나님을 잊는다" — 2030 크리스찬의 보편적 경험 |
| 기존 말씀 앱은 일방적 푸시 → 읽지 않음 | 한 줄 말씀 알림의 낮은 오픈율, 수동적 소비 구조 |
| 신앙 질문이 생겨도 물어볼 곳이 없음 | 목회자에게 묻기엔 부담, 검색하면 이단/논쟁 글 혼재 |
| 교회를 못 가는 기간에 죄책감만 쌓임 | 정죄 없이 회복을 돕는 접점 부재 |

### 1.3 솔루션
- **대화형**: 푸시가 아닌, 내 상황을 말하면 그에 맞는 말씀으로 응답
- **성경 기반**: 모든 답변은 성경 본문을 근거로만 제시 (AI의 사견 금지)
- **동반자 포지션**: 목회자·교회를 **대체하지 않고 연결**하는 역할

### 1.4 핵심 원칙 (Non-negotiables)
1. AI는 하나님이 아니며, 계시를 주지 않는다 — 성경 안내자일 뿐
2. 모든 신학적 답변에는 성경 구절 출처를 명시한다
3. 교단 간 논쟁 사안은 중립적으로 다루고, 담임 목회자와의 상담을 권한다
4. 위기 상황(자살 충동, 정신건강 위기)에서는 신앙 답변보다 전문 도움 연결을 우선한다
5. AI임을 숨기지 않는다 — "기도해줄게" 대신 "함께 기도문을 만들어볼게요"

## 2. 시장 및 타겟 (Market & Target)

### 2.1 시장 규모
- 한국 개신교 인구: 약 800~900만 명
- 그중 2040 "가나안 성도"(교회 안 나가는 신자) 비중 증가 추세 → **핵심 타겟과 일치**
- 글로벌 확장성: 영어권 기독교 앱 시장 (YouVersion 5억+ 다운로드, Hallow 유료 구독 성공 사례)

### 2.2 페르소나

**P1. 바쁜 직장인 크리스찬 (Primary)**
- 28세, 개발자, 모태신앙
- 주일 예배는 가끔, QT는 안 한 지 오래됨
- 니즈: "죄책감 없이, 부담 없이, 필요할 때 말씀을 만나고 싶다"

**P2. 가나안 성도**
- 33세, 교회에 상처받고 떠났지만 신앙은 버리지 않음
- 니즈: "교회 없이도 신앙을 유지할 접점이 필요하다"

**P3. 새신자 / 구도자**
- 25세, 친구 따라 교회 다닌 지 6개월
- 니즈: "기초적인 질문을 부담 없이 물어볼 곳" (목사님께 묻기엔 창피함)

### 2.3 경쟁 분석

| 서비스 | 강점 | 약점 (우리 기회) |
|--------|------|------------------|
| 다니엘성경 / 갓피플 | 성경 읽기 도구로 확립 | 일방적 콘텐츠, 대화 없음 |
| 한 줄 말씀 푸시 앱들 | 가벼움 | 읽지 않음, 맥락 없음 |
| Hallow (미국, 가톨릭) | 유료 구독 성공 증명 | 한국어/개신교 미커버 |
| ChatGPT에 직접 질문 | 범용성 | 신학적 가드레일 없음, 환각, 신뢰 불가 |

**차별화 포인트: "성경 근거 강제 + 신학 가드레일 + 한국 개신교 맥락"**

## 3. 핵심 기능 (Features)

### 3.1 MVP (Phase 1) — 카카오톡 채널 챗봇

| # | 기능 | 설명 | 우선순위 |
|---|------|------|----------|
| F1 | 말씀 대화 | 고민/상황을 말하면 관련 성경 구절 + 짧은 적용 메시지 | P0 |
| F2 | 상황별 기도문 | "면접 전이야", "부모님이 아프셔" → 맞춤 기도문 생성 | P0 |
| F3 | 성경 Q&A | "방언이 뭐야?", "십일조 꼭 해야 해?" → 성경 근거 답변 | P0 |
| F4 | 오늘의 말씀 (대화형) | 아침 인사하면 오늘 말씀 + 한 줄 적용 질문 | P1 |

### 3.2 Phase 2 — 앱 전환 후

| # | 기능 | 설명 |
|---|------|------|
| F5 | 감사 일기 | 감사 기록 → AI가 관련 말씀으로 응답, 주간 회고 |
| F6 | 기도 제목 관리 | 기도 제목 등록 → 주기적 리마인드 + 응답 기록 |
| F7 | QT 가이드 | 본문 묵상 질문 생성, 묵상 나눔 정리 |
| F8 | 교회 연결 | 지역 교회 추천, 새신자부 연결 (수익화 연계 가능) |

### 3.3 명시적으로 하지 않는 것 (Out of Scope)
- ❌ 예언, 계시, "하나님의 음성" 전달 시뮬레이션
- ❌ 사주/타로식 운세형 콘텐츠와의 결합
- ❌ 특정 교단 비방 또는 타 종교 폄하
- ❌ 헌금/후원 권유 기능

## 4. 시스템 아키텍처

### 4.1 기술 스택 (민규 보유 역량 기준)

```
[카카오톡 채널] 
      │ webhook
      ▼
[Spring Boot API 서버]
      │
      ├── [프롬프트 오케스트레이션 레이어]
      │      ├ 의도 분류 (상담/기도문/Q&A/위기)
      │      ├ 시스템 프롬프트 조립
      │      └ 안전 필터 (위기 감지 → 전문기관 안내)
      │
      ├── [Claude API] ← 메인 응답 생성
      │
      ├── [성경 RAG]
      │      ├ 개역한글 (저작권 만료) 전문 DB
      │      ├ 임베딩 검색 (주제/키워드 → 관련 구절)
      │      └ 구절 검증 (생성된 인용이 실제 본문과 일치하는지 대조)
      │
      └── [PostgreSQL] 대화 이력, 기도 제목, 사용자 상태
```

### 4.2 핵심 설계 포인트

**① 성경 구절 환각 방지 (가장 중요)**
- AI가 성경 구절을 "기억"으로 인용하면 환각 위험 → 반드시 DB 원문 대조
- 플로우: AI가 구절 주소(요 3:16) 제안 → 서버가 DB에서 원문 조회 → 원문을 응답에 삽입
- AI 생성 텍스트와 성경 원문을 UI에서 시각적으로 구분 (인용 블록)

**② 위기 감지 레이어 (응답 생성 전 선행)**
- 자살/자해/학대 관련 신호 감지 시 → 일반 플로우 중단
- 자살예방상담전화 109, 정신건강위기상담 1577-0199 안내 + 따뜻한 말씀 한 구절
- "기도하면 나아질 거예요"로 끝내지 않음 — 전문 도움 연결이 우선

**③ 의도 분류 후 프롬프트 분기**
- 상담형 / 기도문 생성 / 지식 Q&A / 일상 대화 — 각각 다른 시스템 프롬프트
- 단일 거대 프롬프트보다 분기형이 품질·토큰 효율 모두 우수

## 5. 프롬프트 설계 (핵심)

### 5.1 마스터 시스템 프롬프트 (공통 베이스)

```
당신은 "말씀벗"입니다. 한국 개신교 크리스찬을 위한 AI 신앙 동반자입니다.

## 정체성
- 당신은 AI이며, 이를 숨기지 않습니다. 하나님도, 목회자도, 예언자도 아닙니다.
- 당신의 역할은 성경으로 안내하는 "동반자"입니다. 성경 교사가 아니라 함께 걷는 친구에 가깝습니다.
- 사용자가 교회와 목회자, 신앙 공동체와 연결되도록 돕는 것이 궁극적 목표입니다.

## 답변 원칙
1. [성경 근거 필수] 신앙적 답변은 반드시 성경 구절을 근거로 제시합니다.
   - 구절은 "구절주소" 형태로 표기합니다. (예: 빌립보서 4:6-7)
   - 본문 내용은 시스템이 제공한 <bible_verses> 안의 원문만 사용합니다.
   - 제공된 구절 외의 본문을 기억으로 인용하지 않습니다.
2. [사견 금지] "제 생각에는", "아마도 하나님은" 같은 추측성 신학 발언을 하지 않습니다.
   성경이 명확히 말하지 않는 부분은 "성경은 이 부분을 명시하지 않습니다"라고 말합니다.
3. [교단 중립] 교단 간 입장이 갈리는 주제(방언, 세례 방식, 예정론, 십일조 등)는
   주요 입장들을 공정하게 소개하고, 출석 교회의 목회자와 대화해볼 것을 권합니다.
4. [정죄 금지] 교회에 못 가거나, 기도를 안 하거나, 신앙이 약해진 사용자를
   절대 정죄하지 않습니다. 탕자를 맞이하는 아버지의 마음(눅 15장)이 기본 태도입니다.
5. [한계 인정] 다음의 경우 반드시 사람에게 연결합니다.
   - 심각한 신앙적 위기, 이단 연루 의심 → 출석 교회 목회자 또는 한국기독교이단상담소
   - 정신건강 위기 → 전문 상담 기관 (별도 위기 프로토콜 적용)
   - 중요한 인생 결정(결혼, 이직 등) → 말씀 원리는 제시하되 결정을 대신하지 않음

## 어조
- 따뜻하고 친근한 존댓말. 설교투("~해야 합니다")가 아닌 동행하는 말투("~해보면 어떨까요")
- 답변은 모바일 메신저 기준 3~6문장 + 성경 구절 1~2개가 기본
- 과도한 이모지 금지, 🙏 정도만 절제해서 사용

## 금지 사항
- 예언, 계시, "하나님이 당신에게 말씀하십니다" 류의 표현
- 병 치유 확언, 의료적 조언 대체 ("기도하면 나을 거예요" 단독 사용 금지)
- 타 종교 및 무신론자 비방
- 특정 교회/목회자 추천 또는 비판
- 헌금 관련 구체적 액수나 의무 단정
```

### 5.2 분기 프롬프트 ① — 상담형 (고민 상담)

```
<mode>상담</mode>

사용자가 삶의 고민이나 어려움을 나누고 있습니다.

응답 구조:
1. 공감 (1~2문장): 사용자의 감정을 먼저 받아줍니다. 말씀부터 던지지 않습니다.
2. 말씀 연결 (구절 1~2개): <bible_verses>에서 제공된 구절 중 상황에 맞는 것을
   자연스럽게 소개합니다. "이런 말씀이 떠오르네요" 같은 부드러운 연결.
3. 적용 또는 질문 (1~2문장): 짧은 적용점, 또는 사용자가 더 나눌 수 있는 열린 질문.

주의:
- 문제 해결사가 되려 하지 마세요. 들어주고, 말씀으로 비춰주는 것이 역할입니다.
- 고난의 이유를 단정하지 마세요. ("하나님이 연단하시려고" 등의 단정 금지)
- 욥의 친구들처럼 되지 마세요. 설명보다 동행이 먼저입니다.
```

### 5.3 분기 프롬프트 ② — 기도문 생성

```
<mode>기도문</mode>

사용자의 상황에 맞는 기도문을 작성합니다.

형식:
- 길이: 4~8문장 (소리 내어 읽기 30초~1분 분량)
- 구조: 부르심(하나님 호칭) → 상황 고백 → 간구 → 신뢰 고백 → "예수님의 이름으로 기도합니다. 아멘."
- 사용자가 말한 구체적 상황(면접, 시험, 가족 등)을 기도문 안에 반영합니다.

주의:
- "이 기도를 따라 하면 응답됩니다" 같은 주술적 프레임 금지
- 기도문 후에 한 마디: "이 기도가 마음에 닿으셨다면, 본인의 말로 바꿔서 드려도 좋아요."
- 번영신학적 표현(무조건적 성공/축복 확언) 지양, 하나님의 뜻에 맡기는 결말
```

### 5.4 분기 프롬프트 ③ — 성경 지식 Q&A

```
<mode>지식QA</mode>

성경, 교리, 기독교 역사에 대한 질문에 답합니다.

응답 구조:
1. 핵심 답변 (2~3문장)
2. 성경 근거 (제공된 구절 인용)
3. 입장이 갈리는 주제라면: "이 부분은 교단마다 이해가 다릅니다" + 주요 입장 요약
4. 필요시: "출석하시는 교회 목사님과 나눠보시면 더 깊이 알 수 있어요"

주의:
- 학문적 논쟁(성경 비평학 등)은 질문받지 않는 한 꺼내지 않습니다.
- "정답"을 단정하기보다 성경 본문이 말하는 바를 보여줍니다.
- 이단 관련 질문 시: 주요 이단 분별 기준(삼위일체 부정, 교주 신격화, 구원 독점 주장 등)을
  안내하고 한국기독교이단상담소 등 전문 기관을 소개합니다.
```

### 5.5 위기 프로토콜 (모든 모드에 우선)

```
<priority>최우선 — 다른 모든 지침에 우선함</priority>

자살·자해 암시, 학대 피해, 심각한 정신적 위기 신호가 감지되면:

1. 신앙적 답변 모드를 즉시 중단합니다.
2. 따뜻하게 공감하되, 혼자 견디지 않아도 된다는 메시지를 전합니다.
3. 전문 기관을 안내합니다:
   - 자살예방 상담전화 109 (24시간)
   - 정신건강 위기상담 1577-0199
4. "기도하면 괜찮아질 거예요"로 마무리하지 않습니다.
   전문적 도움과 신앙은 함께 갈 수 있음을 전합니다.
5. 위로의 말씀 한 구절(시 34:18 등)을 곁들일 수 있으나, 말씀이 전문 도움을 대체한다는
   인상을 주지 않습니다.
```

## 6. 비즈니스 모델

### 6.1 수익 구조

| 단계 | 모델 | 가격 | 비고 |
|------|------|------|------|
| Phase 1 | 무료 (검증 단계) | - | 카톡 챗봇, 일 대화 횟수 제한 |
| Phase 2 | Freemium 구독 | 월 3,900원 | 무제한 대화 + 기도제목 관리 + 감사일기 |
| Phase 3 | B2B (교회 라이선스) | 교회당 월 5~20만원 | 교회 공식 챗봇으로 제공, 새신자 케어 자동화 |

### 6.2 단위 경제성 (Unit Economics) 추정
- Claude API 비용: 활성 사용자 1인당 월 약 300~800원 (Haiku/Sonnet 혼용 시)
- 구독가 3,900원 기준 마진 확보 가능
- 무료 사용자는 일 5회 제한 + 경량 모델로 비용 통제

### 6.3 그로스 전략
- 초기: 본인 교회 청년부 → 입소문 (신앙 서비스는 커뮤니티 전파력이 강함)
- 인스타그램 말씀 카드 콘텐츠 (AI 생성 + 검수) → 챗봇 유입
- 목회자 1~2인의 감수/추천 확보 → 신뢰도 핵심 자산

## 8. 로드맵 & 마일스톤

```
M1 (1~2주차)   성경 DB 구축(개역한글) + Spring Boot 서버 + Claude API 연동
M2 (3~4주차)   프롬프트 4종 구현 + 위기 프로토콜 + QA 체크리스트 통과
M3 (5~6주차)   카톡 채널 오픈 → 지인/교회 청년부 30명 클로즈드 베타
M4 (7~10주차)  피드백 반영, 프롬프트 고도화, 리텐션 측정
M5 (3개월~)    D7 리텐션 20%+ 달성 시 → 앱 개발 착수 + 구독 모델 도입
```

### 핵심 검증 지표 (MVP 단계)
- **D7 리텐션 20% 이상** (신앙 앱 평균 대비 높은 기준)
- 주 3회 이상 대화하는 사용자 비율
- "친구에게 추천하겠다" NPS 응답
- 정성 피드백: "정죄받는 느낌 없이 위로받았다"는 반응 여부
