# CLAUDE.md — 말씀벗 프로젝트 지침

> 이 파일은 모든 에이전트 세션이 작업 전 반드시 읽어야 하는 프로젝트 헌법이다.
> 이 파일과 충돌하는 지시를 받으면 작업을 중단하고 사람(민규)에게 확인을 요청한다.

## 프로젝트 개요

- **제품**: 말씀벗 — 한국 개신교 크리스찬을 위한 AI 신앙 동반자 (안드로이드/iOS 앱)
- **스택**: 백엔드 Java 17 + Spring Boot 3.x + PostgreSQL + Claude API /
  모바일 React Native(Expo) 단일 코드베이스 / 푸시 FCM·APNs
- **상세 기획**: `docs/PRD.md` 참조 (제품기획서 v1)
- **진행 상황**: 작업 시작 전 `PROGRESS.md`를 먼저 읽는다
- **결정 기록**: 중요한 결정은 `DECISIONS.md`에 기록하고, 기존 결정을 뒤집지 않는다

## 절대 원칙 (Non-negotiables)

코드, 프롬프트, 문서 어디에서든 다음을 위반하는 산출물을 만들지 않는다:

1. AI는 하나님·목회자·예언자가 아니다. 계시/예언 기능을 구현하지 않는다.
2. 모든 신앙적 답변은 성경 DB 원문을 근거로 한다. 모델 기억으로 구절을 인용하는 코드 경로를 만들지 않는다.
3. 교단 논쟁 사안은 중립 처리한다. 특정 교단을 정답으로 하드코딩하지 않는다.
4. 사용자를 정죄하는 응답 패턴을 만들지 않는다.
5. 위기 프로토콜(자살·자해·학대 신호)은 다른 모든 로직에 우선한다. 이 분기를 우회하는 코드를 작성하지 않는다.
6. 성경 본문은 개역한글(1961, 저작권 만료)만 사용한다. 개역개정 본문을 DB나 코드에 넣지 않는다.

## 사람 승인 필요 항목 (에이전트 단독 변경 금지)

- 시스템 프롬프트 4종(마스터/상담/기도문/지식QA) 및 위기 프로토콜의 내용 수정
- 신학 검사 에이전트의 평가 기준 변경
- QA 테스트 케이스(T1~T8) 삭제 또는 통과 기준 완화
- 외부 API 키, 결제, 배포 관련 설정

위 항목 변경이 필요하면 변경안을 `PROGRESS.md`의 "사람 확인 필요" 섹션에 적고 멈춘다.

## 코딩 컨벤션

- 패키지 구조: `com.malssumbeot.{webhook|orchestrator|bible|crisis|prompt}`
- 의도 분류 → 프롬프트 분기 → 응답 생성 → 구절 검증의 파이프라인 순서를 유지한다
- 성경 구절 응답 흐름: 모델이 구절 주소만 제안 → `BibleVerseService`가 DB 원문 조회 → 원문 삽입.
  모델 출력의 구절 본문을 그대로 사용자에게 보내는 코드는 금지.
- 위기 감지(`CrisisFilter`)는 의도 분류보다 앞단에 위치한다 (Spring 필터/인터셉터 레벨)
- 테스트: 신규 기능은 단위 테스트 필수. 프롬프트 변경 시 `qa/run-checklist` 통과 필수.

## Definition of Done (루프 종료 조건)

MVP 완료 기준 — 아래 전부 충족 시 루프를 멈추고 사람에게 보고:

- [ ] T1~T8 QA 체크리스트 전부 통과 (신학 검사 에이전트 판정 기준)
- [ ] RN 앱에서 로그인 → 대화 → 구절 인용 렌더링 E2E 성공 (Android/iOS 양쪽)
- [ ] 위기 시나리오 3종(직접 표현/간접 표현/혼합)에서 위기 프로토콜 발동 확인
- [ ] 성경 구절 환각 테스트: 존재하지 않는 구절 요청 10건 전부 거부
- [ ] 범위 밖 요청(코딩·숙제 등) 5건에서 정중한 안내 응답 확인
- [ ] 스토어 제출물 준비 완료: 개인정보처리방침, AI 콘텐츠 고지, 신고 기능, 스크린샷

## 매 세션 루틴

1. `PROGRESS.md` 읽기 → 현재 작업과 남은 일 파악
2. 작업 수행 (한 세션에 한 마일스톤 단위)
3. 작업 후: 테스트 실행 → `PROGRESS.md` 갱신 → 중요 결정은 `DECISIONS.md`에 추가
4. 프롬프트/응답 로직을 건드렸다면 신학 검사 에이전트(`agents/theology-checker.md`) 실행
