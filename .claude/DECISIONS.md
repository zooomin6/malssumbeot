# DECISIONS.md — 말씀벗 결정 기록

> 한 번 기록된 결정은 에이전트가 임의로 번복하지 않는다.
> 번복이 필요하면 "재검토 요청" 섹션에 사유를 적고 사람 승인을 기다린다.

## 확정된 결정

### D-001. 성경 역본: 개역한글 (1961)
- 날짜: 2026-06-12
- 이유: 저작권 만료로 상업 서비스에 자유롭게 사용 가능. 개역개정은 대한성서공회
  라이선스 필요 → 수익화 검증 후 협의.
- 영향: 성경 DB, 모든 구절 인용

### D-002. MVP 플랫폼: 안드로이드 + iOS 네이티브 앱 (React Native) ★개정
- 날짜: 2026-06-12 (당일 개정, 민규 승인)
- 변경 전: 카카오톡 채널 챗봇 → 변경 후: RN(Expo) 기반 크로스플랫폼 앱
- 이유: 푸시 알림·감사일기·기도제목 등 핵심 경험을 처음부터 앱으로 제공,
  구독 결제 구조에 유리. 검증 기간이 2~3개월로 늘어나는 트레이드오프는 수용.
- 영향: webhook 모듈 → REST API + 인증(소셜 로그인) + 푸시(FCM/APNs)로 대체.
  의도 분류·성경 RAG·위기 필터·프롬프트는 전부 재사용.

### D-007. 앱 기술 스택: React Native (Expo) 단일 코드베이스
- 날짜: 2026-06-12
- 이유: 안드로이드/iOS 동시 출시, 채팅 UI 생태계 성숙, 협업 가능한 RN 경험자 존재.
- 영향: 모바일 레포 분리 (backend / mobile), Expo 빌드 파이프라인

### D-008. 자유 대화 기능 추가 — "신앙 중심 + 일상 동행" 하이브리드
- 날짜: 2026-06-12
- 이유: ChatGPT와 범용성으로 경쟁하지 않음. 일상 대화는 친구처럼 받아주되
  말씀을 강요하지 않고, 코딩·숙제 등 범위 밖 요청은 정중히 안내.
- 영향: 의도 분류에 "일상 대화"·"범위 밖" 카테고리, 모델 라우팅(일상=경량 모델)

### D-003. 성경 구절은 모델 기억이 아닌 DB 원문만 사용
- 날짜: 2026-06-12
- 이유: 구절 환각은 이 서비스의 치명적 리스크. 모델은 구절 주소만 제안하고
  서버가 원문을 조회·삽입한다.
- 영향: BibleVerseService 파이프라인, theology-checker C1 기준

### D-004. 위기 프로토콜은 모든 로직에 우선
- 날짜: 2026-06-12
- 이유: 힘든 사용자가 많이 유입될 서비스 특성상 안전이 신학·기능보다 우선.
- 영향: CrisisFilter를 의도 분류 앞단에 배치, 우회 불가 설계

### D-005. 만드는 에이전트와 검사하는 에이전트 분리
- 날짜: 2026-06-12
- 이유: 자기 산출물 자기 검증은 신뢰 불가. 신학 검사는 독립 에이전트가 비판적으로 수행.
- 영향: agents/theology-checker.md, 머지 차단 규칙

### D-006. 프롬프트·신학 기준 변경은 사람 승인 필수
- 날짜: 2026-06-12
- 이유: 루프 자동화 범위에서 신학적 판단을 제외. 에이전트는 제안까지만.
- 영향: CLAUDE.md "사람 승인 필요 항목"

### D-009. 백엔드 빌드 도구: Maven (wrapper), Spring Boot 3.5.15
- 날짜: 2026-06-12
- 이유: 개발 머신에 Gradle/Maven 미설치 + 로컬 JDK 25 환경에서 구버전 Gradle 호환 리스크
  → Maven wrapper가 JDK 버전에 무관하게 안정적. Boot 버전은 헌법 명세(3.x)의 최신 패치
  3.5.15로 고정 (Initializr 기본값 4.x는 헌법 위반이라 배제).
- 영향: backend/pom.xml, mvnw 기반 빌드·테스트 파이프라인
