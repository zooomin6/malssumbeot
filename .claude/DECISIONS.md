# DECISIONS.md — 말씀벗 결정 기록

> 한 번 기록된 결정은 에이전트가 임의로 번복하지 않는다.
> 번복이 필요하면 "재검토 요청" 섹션에 사유를 적고 사람 승인을 기다린다.

## 확정된 결정

### D-001. 성경 역본: 개역한글 (1961)
- 날짜: 2026-06-12
- 이유: 저작권 만료로 상업 서비스에 자유롭게 사용 가능. 개역개정은 대한성서공회
  라이선스 필요 → 수익화 검증 후 협의.
- 영향: 성경 DB, 모든 구절 인용

### D-002. MVP 플랫폼: 안드로이드 + iOS 네이티브 앱 (React Native) ★개정
- 날짜: 2026-06-12 (당일 개정, 민규 승인)
- 변경 전: 카카오톡 채널 챗봇 → 변경 후: RN(Expo) 기반 크로스플랫폼 앱
- 이유: 푸시 알림·감사일기·기도제목 등 핵심 경험을 처음부터 앱으로 제공,
  구독 결제 구조에 유리. 검증 기간이 2~3개월로 늘어나는 트레이드오프는 수용.
- 영향: webhook 모듈 → REST API + 인증(소셜 로그인) + 푸시(FCM/APNs)로 대체.
  의도 분류·성경 RAG·위기 필터·프롬프트는 전부 재사용.

### D-007. 앱 기술 스택: React Native (Expo) 단일 코드베이스
- 날짜: 2026-06-12
- 이유: 안드로이드/iOS 동시 출시, 채팅 UI 생태계 성숙, 협업 가능한 RN 경험자 존재.
- 영향: 모바일 레포 분리 (backend / mobile), Expo 빌드 파이프라인

### D-008. 자유 대화 기능 추가 — "신앙 중심 + 일상 동행" 하이브리드
- 날짜: 2026-06-12
- 이유: ChatGPT와 범용성으로 경쟁하지 않음. 일상 대화는 친구처럼 받아주되
  말씀을 강요하지 않고, 코딩·숙제 등 범위 밖 요청은 정중히 안내.
- 영향: 의도 분류에 "일상 대화"·"범위 밖" 카테고리, 모델 라우팅(일상=경량 모델)

### D-003. 성경 구절은 모델 기억이 아닌 DB 원문만 사용
- 날짜: 2026-06-12
- 이유: 구절 환각은 이 서비스의 치명적 리스크. 모델은 구절 주소만 제안하고
  서버가 원문을 조회·삽입한다.
- 영향: BibleVerseService 파이프라인, theology-checker C1 기준

### D-004. 위기 프로토콜은 모든 로직에 우선
- 날짜: 2026-06-12
- 이유: 힘든 사용자가 많이 유입될 서비스 특성상 안전이 신학·기능보다 우선.
- 영향: CrisisFilter를 의도 분류 앞단에 배치, 우회 불가 설계

### D-005. 만드는 에이전트와 검사하는 에이전트 분리
- 날짜: 2026-06-12
- 이유: 자기 산출물 자기 검증은 신뢰 불가. 신학 검사는 독립 에이전트가 비판적으로 수행.
- 영향: agents/theology-checker.md, 머지 차단 규칙

### D-006. 프롬프트·신학 기준 변경은 사람 승인 필수
- 날짜: 2026-06-12
- 이유: 루프 자동화 범위에서 신학적 판단을 제외. 에이전트는 제안까지만.
- 영향: CLAUDE.md "사람 승인 필요 항목"

### D-009. 백엔드 빌드 도구: Maven (wrapper), Spring Boot 3.5.15
- 날짜: 2026-06-12
- 이유: 개발 머신에 Gradle/Maven 미설치 + 로컬 JDK 25 환경에서 구버전 Gradle 호환 리스크
  → Maven wrapper가 JDK 버전에 무관하게 안정적. Boot 버전은 헌법 명세(3.x)의 최신 패치
  3.5.15로 고정 (Initializr 기본값 4.x는 헌법 위반이라 배제).
- 영향: backend/pom.xml, mvnw 기반 빌드·테스트 파이프라인

### D-010. 의도 분류 모델: claude-haiku-4-5 (경량)
- 날짜: 2026-06-12
- 이유: PRD §6.2 단위 경제성("Haiku/Sonnet 혼용")과 모델 라우팅 방침(일상·분류=경량)을 따름.
  분류는 단일 라벨 출력이라 경량 모델로 충분. 응답 생성용 상위 모델은 모델 라우팅
  작업(다음 작업 #4)에서 확정한다.
- 영향: IntentClassifier, application.yml `malssumbeot.anthropic.classifier-model`

### D-011. 위기 라벨은 부분 일치 우선 파싱 (위기 강등 방지)
- 날짜: 2026-06-12
- 이유: theology-checker 판정(2026-06-12, critical) — 모델이 위기를 감지하고도 출력
  형식을 어기면(설명 덧붙임 등) 정확 일치 파싱에 실패해 폴백("상담")으로 강등되는
  경로가 절대 원칙 5(위기 분기 우회 금지)에 저촉. 분류기 출력에 "위기"가 포함되면
  형식과 무관하게 CRISIS로 처리한다.
- 영향: IntentClassifier.classify, D-004 강화. 해석 불가 출력의 폴백은 상담 유지
  (시스템 프롬프트 5종 탑재 시 모든 모드에 위기 프로토콜 내장으로 추가 완화 예정)

### D-012. 위기 감지 1차는 결정론적 패턴 + 세션 sticky 상태
- 날짜: 2026-06-12
- 이유: LLM 분류는 장애·형식 위반·비결정성 리스크가 있어 위기 감지의 단독 수단이
  될 수 없다. 1차는 정규식 패턴(공백 정규화 후 부분 매칭, 재현율 우선), LLM 분류는
  2차 방어선. theology-checker 지적(멀티턴 위기 컨텍스트 휘발)을 반영해 위기 감지 후
  30분(설정값)간 세션을 위기 상태로 유지한다(sticky). 패턴 목록과 유지 시간은
  사람 검토 대상 (PROGRESS "사람 확인 필요").
- 영향: com.malssumbeot.crisis 패키지, crisis-patterns.txt, REST API 인터셉터 배선(예정)

### D-013. 모델 라우팅: 신앙=Sonnet, 일상·범위밖=Haiku
- 날짜: 2026-06-12
- 이유: PRD §6.2 단위 경제성(Haiku/Sonnet 혼용). 신앙 상담·지식QA·기도문·위기는 품질이
  중요해 상위 모델(claude-sonnet-4-6), 일상 대화·범위 밖 안내는 경량 모델(claude-haiku-4-5).
  분류기는 D-010대로 Haiku 유지.
- 영향: ModelRouter, application.yml(faith-model/casual-model), 토큰 비용 통제

### D-014. 환각 구절은 본문에서 제거 후 전송 (재생성으로도 안 잡히면)
- 날짜: 2026-06-12
- 이유: theology-checker 2회차 판정 — 재생성 후에도 미검증(존재하지 않는) 구절 주소가
  응답 본문에 남으면 '그대로 전송'하는 것이 절대 원칙 2 및 T8 '거부' 기준 위반.
  인용 블록 미첨부만으로는 프로즈에 박힌 환각 주소 노출을 막지 못함.
- 결정: 재생성 후에도 미검증 주소가 남으면 그 주소가 든 문장을 본문에서 제거
  (stripSentencesContaining). 제거 후 본문이 비면 결정론적 안전 문구로 대체.
  미검증 주소는 사용자 본문에서 사라지고 모니터링용 목록(unverifiedReferences)에만 남는다.
- 영향: ChatOrchestrator.generate. 알려진 한계(장 단위 인용 미스캔)는 다음 작업 #3로 분리.

### D-015. CLAUDE.md 규칙을 2층으로 구별 + 개발 작업 규칙 추가
- 날짜: 2026-06-27 (민규 지시)
- 이유: 헌법이 "제품 규칙(말씀벗이 응답할 때)"과 "개발 작업 규칙(코딩할 때)"을
  섞고 있었음. 민규가 둘을 구별하고, LLM 코딩 실수를 줄이는 행동 양식 4종
  (코딩 전 생각 / 단순함 우선 / 외과수술적 변경 / 목표 주도 실행)을 제공.
- 결정: 절대 원칙(제품 규칙)은 그대로 두고, "개발 작업 규칙" 섹션을 CLAUDE.md에
  신설. 로그인 등 신앙과 무관한 기능 구현에도 이 섹션이 적용된다.
- 영향: CLAUDE.md "개발 작업 규칙" 섹션. 향후 모든 코딩 작업의 행동 기준.
